Getting Started

1 · Clone & install

bash 
# Clone the repository
git clone https://github.com/your-org/promptforge-ai-saas.git
cd promptforge-ai-saas

# Install dependencies (uses package-lock.json)
npm ci

2 · Configure environment

bash 
cp .env.example .env.local

Edit .env.local with your values — see the Environment Variables section below.

3 · Run locally

bash 
npm run dev

Open http://localhost:3000 in your browser.

Node.js requirement: This project requires Node.js 20 or later. Run node -v to confirm your version.

Tailwind Design Tokens

The project ships three custom color families defined in tailwind.config.ts.

tailwind.config.ts · colors 
// ink — neutral grays
ink-50:  #f7f8fa   → page background
ink-100: #ebeef2   → card surfaces
ink-300: #b7c0cc   → borders
ink-500: #657184   → muted text
ink-700: #333b49   → body text
ink-900: #11151c   → headings

// accent — teal/green (primary brand)
accent-400: #35cfa1  → hover states
accent-500: #12b886  → primary actions
accent-600: #0b946d  → pressed / border

// ember — warm amber (highlights)
ember-100: #fff0d9  → warning bg
ember-400: #ffad4d  → highlight text
ember-500: #f58b22  → badge / tag

Custom utilities also include shadow-soft, shadow-panel, and the animate-fade-up / animate-pulse-soft keyframe animations.

Environment Variables

Create a .env.local file at the project root. Never commit secrets to version control — .env*.local is already in .gitignore.

environment
Variable Purpose Example
NEXT_PUBLIC_APP_URL required Public base URL of your deployment http://localhost:3000
ANTHROPIC_API_KEY required Anthropic API key for AI features sk-ant-…
DATABASE_URL required PostgreSQL / Neon / PlanetScale URL postgres://…
NEXTAUTH_SECRET required Random secret for session signing openssl rand -base64 32
STRIPE_SECRET_KEY optional Stripe key for billing features sk_test_…
RESEND_API_KEY optional Email delivery via Resend re_…
Security note: Variables prefixed with NEXT_PUBLIC_ are bundled into the client and visible in the browser. Never put secrets there.

Contributing

Contributions, bug reports, and feature requests are welcome. Please open an issue before submitting a pull request for significant changes.

bash · workflow 
# 1. Fork the repo, then:
git checkout -b feat/your-feature-name

# 2. Make your changes and verify lint passes
npm run lint

# 3. Confirm the production build succeeds
npm run build

# 4. Push and open a Pull Request
git push origin feat/your-feature-name
Commit style: Use conventional commits — feat:, fix:, chore:, docs: — so the changelog stays readable.
